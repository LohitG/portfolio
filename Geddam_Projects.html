Black Jack

AP Computer Science Applications - 04/19 to 05/19

In AP Computer Science Applications, we were assigned a fourth quarter project. Our task was to create a single player card game. This theme was based on our assignments, which had been based on card games and artificial intelligence algorithms. We could base our project on a variety of card games, but Black Jack and Solitaire were the highly recommended ones. We decided to do our project on Black Jack because we had a good amount of experience with Black Jack and it seemed easier to implement a game based on Black Jack. The hardest part of the project was implementing the Ace's ability to act as a 1 or a 11. However we realized that if the total was under 21, both the user would want it to be an 11, but when their total was over 21, they would want it to be a 1. Using this information, we implemented a simple conditional that automatically changed the value of the Ace for both the player and the computer. We were not able to add additional features but we were able to gracefully implement the features of starting a new game, asking the dealer for another card, and "standing", which meant the player was ready to best the dealer's total. MessageHelper

AP Computer Science Applications - 02/19 to 03/19

For our Quarter 3 project in AP Computer Science Applications, we were assigned with the task of making a mobile application that used elements learned in class such as a database, automation, or initialization. We were allowed to choose the language that would be used for the application. Android Studio, where we did most of our assignments, was a popular IDE among our peers, as they chose to program in Java. However due to the unnecessary complication of Android Studio, our team opted to program in React Native. MessageHelper was meant to help people receive and reply to texts while they were busy or incapable. MessageHelper utilized speech to text and text to speech in order to achieve this. We wanted to be able to store user preferences and allow users to create accounts but we didn't know which database service to use. After much research, it was found that Google's FireBase was the easiest to implement. Using FireBase, each user was able to have their own specialized account. We were able to set up an automation that allowed new users to create new accounts. This project was largely built off of my previous CSE Project, which can be seen below. 2.3.2 Bridge Design

Principles of Engineering - 03/09/18 to 03/20/18

In project 2.3.2 Bridge Design, I was assigned to design a stable, cost efficient bridge for our hypothetical client, American Bridge Company. I completed this project with my classmate, Ayush Roy. As a duo, we created this Gantt Chart so that we could work more time effectively. In order to make sure we completed the project to the fullest extent, we constantly referred to the rubric. We started the project by researching the cost and strength differences between the different materials, cross sectional areas, and solid verus hollow members. We discovered that quenched steel was the strongest and was the best material since it was not too expensive for its quality. We ended up using this material in our bridge. The cross sectional area gave more strength as it was increased, but increasing the area dramatically increased the price. We knew that the area may be situational among members. In our research, we also identified that solid bars could take tension the better than compression while hollow bars did exactly the opposite. We then brainstormed two ideas each and discussed them. We found that the Warren bridge had the most stability and value so we chose it to improve on. We then shared our design with the class and looked at our peers' bridges. Our documentation can be accessed here.
