1.1.6 Compound Machine

Principles of Engineering - 08/30/17 to 09/11/17

The 1.1.6 Compound Machine was the first project of POE. The goal of the project was to incorporate four different types of simple machines that worked together in order to to accomplish a task. The main constraint was that only one source of human force could be applied. There was an intial brainstorming session in which each group member drew up their own ideas. We then shared ur ideas with one and another before coming to a joint agreement on a machine what would mark papers. I was named the project manager so I assigned roles and created a Gantt Chart so that we would know when things needed to be done by so that we would have a smooth project. However when things went wrong, improvisation was needed so the Gantt chart was not followed exactly. It still served its purpose of providing guidlines. All of our modifications were recorded in our documentation, which was built to the requirements of the rubric. It was an overall success and as well as a project I enjoyed very much.

Text Adventure Project

Computer Science Engineering

The Text Adventure Project was my third project in CSE. We had to make a interactive story, similar to my Scratch project. Except in this project, only text was allowed since we were using python to code the project. Just like our other projects, we had a brainstorming session in which we came up with different themes and stories to use in our project. Very identical to our Scratch project. After considering our ideas, we narrowed the choices down to two and requested advice from our peers. After considering their input, we decided to go with a hijack theme for our story. We split up the programming evenly so that I coded one story line and he coded the other story line. When the story lines mixed, we collaborated together and finished the project. Once our content was finished, we tried to add a feature that would let you redo a decision is you typed in something invalid. While did manage a replay button, we were not able to implement our original improvement. Our overall work ethic was impressive and we were able to achieve a great deal together.

MIT App Inventor Project

Computer Science Engineering

The MIT App Inventor project was my second project in CSE. In this project, we were instructed to select a base app that we would later build upon. To get ourselves familiarized with MIT App Inventor, we did a few activities in which we were able to get to acknowledge the numerous functionalities that were offered to us by MIT App Inventor. After much thought, my partner and I chose an app that automated text messages for you, making it so that you do not need to reply to messages when your busy. It also read out teh text. But this was all the base app gave us. We did a lot more than that. Afterwe finished our base app, we brainstormed and got peer feedback very similar to my previous Scratch Project. We had many ideas on what to do and in the end me were able to accomplish all of them. We started off my make the User Interface much better. Now it was much more pleasant to the eye. Furthermore, we made it so that our app replies to calls as well as texts. Additionally, we added a voice recognizer that converted what you said into a text. Finally, after much debate we also added a transalator that could transalate your text into seven different languages. Developing the app was a great experience,especially when we were able to see it in action.
